# Quality Checklist for Helper Prompt Outputs

Use this checklist before pasting helper output into a controlled Word document.

## Basic wording check

- [ ] The revised text is clearer than the original.
- [ ] The revised text uses formal QMS tone.
- [ ] The revised text is not too wordy.
- [ ] The revised text avoids conversational wording.
- [ ] The revised text uses approved company terminology.

## Meaning preservation check

- [ ] No required action was made optional.
- [ ] No optional action was made mandatory.
- [ ] No responsibility changed.
- [ ] No approver changed.
- [ ] No reviewer or verifier changed.
- [ ] No record or evidence requirement was removed.
- [ ] No retention requirement changed.
- [ ] No process step was added without source support.
- [ ] No process step was removed.
- [ ] No process sequence changed.
- [ ] No scope boundary changed.

## Stop words to check carefully

Review the output carefully if any of these words were added, removed, or changed:

```text
shall
must
should
may
will
can
required
responsible
approve
review
verify
inspect
authorize
retain
maintain
record
evidence
customer
regulatory
statutory
```

## Formatting-ready check

- [ ] Heading text is short enough for the section heading.
- [ ] Procedure steps use consistent sentence structure.
- [ ] Table cell text is concise.
- [ ] Activity map labels are short enough for diagram boxes.
- [ ] Acronyms are defined or flagged.

## Approval check

Send the change for human review if any box below is checked:

- [ ] The helper says meaning impact is Medium or High.
- [ ] The helper asks a process owner question.
- [ ] The helper asks a Quality question.
- [ ] The helper changed an obligation word.
- [ ] The helper changed a role, record, approval, scope, or sequence.
- [ ] The helper added information that was not in the original source.

## Final habit

Keep the original text, revised text, and approval decision together in your project notes.
