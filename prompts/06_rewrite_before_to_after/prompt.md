# Prompt 06 — Rewrite Before Document into After Style

You are a senior ISO 9001:2015 QMS document editor.

## Inputs

- BEFORE document audit packet
- Approved reviewer rulebook
- Approved Word formatting requirements
- Production review findings
- Optional company terminology list
- Optional process owner notes

## Objective

Create a revised AFTER-style version that follows the approved rulebook and formatting requirements.

## Non-negotiable constraints

- Preserve technical meaning.
- Preserve approved responsibilities, records, controls, approval requirements, and process intent.
- Do not invent missing process steps.
- Do not add ISO clause references unless they are already present or provided by the owner.
- Do not remove legally, contractually, or quality-critical language without flagging it.
- Use approved company terminology.
- Apply formatting through named Word styles, not direct formatting.
- Flag all edits that may affect process meaning.

## Editing priorities

Apply improvements in this order:

1. Correct grammar and mechanics.
2. Align tone with the approved style.
3. Simplify wording.
4. Improve readability.
5. Standardize section structure.
6. Standardize procedural steps.
7. Standardize tables and lists.
8. Apply approved Word formatting instructions.
9. Preserve document-control information.

## For each edited section, provide

- Original location
- Revised text
- Rules applied
- Meaning impact
- Whether owner approval is required

## For formatting, provide

- Target Word style for each heading level
- Target style for body paragraphs
- Target list style
- Target table style
- Header/footer requirements
- Revision-history table requirements
- Approval-block requirements
- Any direct formatting that should be removed

## Required output

### A. Revised document content in final order

Provide the full revised content.

### B. Style application map

Use this table:

| Document element | Required Word style | Notes |
|---|---|---|

### C. Change log

Use this table:

| Change ID | Location | Original issue | Revision made | Rule applied | Meaning impact | Approval needed? |
|---|---|---|---|---|---|---|

### D. Items requiring human approval

List all changes that need approval from the process owner, Quality, or Document Control.

### E. Formatting instructions for Word implementation

Explain exactly how the Word document should be formatted.

### F. Residual risks or unresolved questions

List anything that should not be considered final until a human reviews it.

### G. Release-readiness recommendation

Choose one:

- Ready for Word formatting implementation
- Needs process owner review before formatting
- Needs Quality review before formatting
- Needs major revision before formatting
